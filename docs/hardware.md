## ESP32-S3 车载控制器接线（Freenove FNK0085）

### 板载约束（Freenove FNK0085）

| 引脚范围 | 状态 | 说明 |
|---------|------|------|
| GPIO 26-37 | 不可用 | 连接板载 SPI Flash，禁止复用 |
| GPIO 19/20 | 不可用 | USB D+/D- |
| GPIO 43/44 | 不可用 | USB-Serial 串口 |
| GPIO 45/46 | 受限 | Strapping 引脚，启动状态需保持 |
| GPIO 0-18 (除 14) | 摄像头占用 | OV2640 引脚 |
| GPIO 1/2/21/38-42 | 可用 | 电机驱动 + 编码器 |

## ESP32-C6 接收器 Dongle 接线

### 电脑端连接

```
ESP32-C6 接收器
├── USB-C ──────► 电脑 USB 端口
│   ├── 供电: 5V
│   ├── 数据: USB 串口
│   └── 地线: GND
│
└── 无线通信: WiFi UDP (2.4GHz)
    ├── SoftAP: SSID/密码见本地 firmware/libraries/wireless_protocol/src/wifi_credentials.h（模板见 wifi_credentials.example.h），静态 IP 192.168.4.1
    ├── 控制端口 9000: 接收电脑命令并转发给车载 S3
    ├── 遥测端口 9001: 接收 S3 测速、链路状态并转发给电脑
    └── 视频端口 9002: 接收 S3 视频帧分片并转发给电脑
```

### 链路状态上报协议

接收器 Dongle 通过 USB 串口向电脑端上报链路状态：

- **触发方式 1**：收到 `'P'` 命令时立即上报
- **触发方式 2**：每 5 秒主动上报一次（周期性心跳）

**JSON 格式**：
```json
{"t":"link","dongle":"ok","car_paired":true,"last_odom_ms":12345}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `t` | string | 消息类型，固定为 `"link"` |
| `dongle` | string | Dongle 自身状态，固定为 `"ok"` |
| `car_paired` | bool | 车载 S3 是否已通过 WiFi UDP 连接到 C6（收到过 S3 数据） |
| `last_odom_ms` | number | 上次收到车载数据的毫秒时间戳（相对启动） |

电脑端后端解析此 JSON 并通过 WebSocket 推送给前端，前端显示链路状态：`探测中` → `Dongle 已连接` → `小车 WiFi 在线`。

- 持续功率: ~5W

- 推荐电池: 7.4V 2S 3000mAh LiPo

## 注意事项

### 电源
1. **必须共地**: 所有设备的地线必须连接在一起
2. **电机隔离**: L298N 的电机电源和逻辑电源分开
3. **滤波电容**: 在电机电源处添加 100μF 电容
4. **保险丝**: 在电池正极添加 5A 保险丝

### 信号
1. **电机 PWM**: 频率 1kHz-20kHz
2. **编码器中断**: GPIO 1/2 FALLING 触发，线长尽量短
3. **无线天线**: ESP32-S3 的天线区域避免金属遮挡
4. **摄像头排线**: OV2640 的排线尽量短，避免干扰（Freenove 板载排线已优化）

### 板载约束（Freenove FNK0085）
1. **GPIO 26-37 禁用**: 连接板载 SPI Flash，复用会导致启动失败
2. **GPIO 19/20 禁用**: USB D+/D-
3. **GPIO 43/44 禁用**: USB-Serial 串口（下载/调试）
4. **GPIO 45/46 受限**: Strapping 引脚，启动状态需保持，运行时可复用但需谨慎

### 安全
1. **紧急停止**: 保留物理急停按钮（前端长按 500ms 触发软件急停）
2. **电源开关**: 添加总电源开关
3. **过热保护**: L298N 添加散热片
4. **反接保护**: 电池接口添加反接保护二极管
